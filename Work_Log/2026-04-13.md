## 확정된 설계 결정

### 시스템 목적 재정의
- Asset Generator는 에디터가 아니다
- 인게임 에셋 조합 렌더링 확인 + 마음에 드는 조합을 고유 캐릭터로 등록하는 도구

### 캐릭터 구성 확정
- Portrait(512×640) + Field Sprite(80×96) 동시 생성, 동일 Character Data 사용
- Field Sprite는 인게임에서 2× 스케일로 표시 (160×192)
- 참고 스타일: 창세기전 3

### Layer System
- Beard 레이어 추가 확정 (Mouth 위, Hair_Back 아래)
- 최종 순서: BodyBase → FaceBase → Eyes → Nose → Mouth → Beard → Hair_Back → Hair_Side → Hair_Front → Armor → Helmet_Back → Helmet_Front

### Anchor System
- FaceTemplate 이미지에 약속된 색상으로 1픽셀 마커 직접 찍는 방식 확정
- Eyes/Nose/Mouth: X,Y 모두 마커 위치 사용
- Hair/Helmet: Y만 사용, X는 캔버스 중앙 고정
- 마커 색상표: Eye_L=#FF0000, Eye_R=#00FF00, Nose=#0000FF, Mouth=#FFFF00, Hair_Front=#FF00FF, Hair_Side/Back=#00FFFF, Helmet=#FF8000

### Helmet Types 재정의
- 기존 OPEN/HALF/CLOSED 폐기
- 신규: TIARA(헤어 유지, 얼굴 유지) / HAT(헤어 유지, 얼굴 유지) / MASK(헤어 유지, 코입수염 hide) / HELM(헤어 hide/swap, 코입수염 hide)

### Color System 확정
- 파츠 에셋: 그레이스케일 베이스로 제작
- 색상 적용 방식: 그라디언트 매핑 (주 컬러 1개 → 하이라이트/그림자 자동 계산)
- Scale/Color 값은 캐릭터 데이터가 소유, 파츠 에셋 자체는 항상 스케일 1

## 기술적 발견 (중요)

### Krita Multiply 블렌드 모드 사용 금지
- Multiply 블렌드로 색을 올리면 **하이라이트가 적용되지 않는다**
- 올바른 방법: **필터 > 지도 > 그라디언트 맵**

### Godot 그라디언트 매핑 셰이더 작성 완료
```glsl
shader_type canvas_item;

uniform vec4 main_color : source_color;

void fragment() {
    float brightness = texture(TEXTURE, UV).r;

    vec4 shadow    = vec4(main_color.rgb * 0.4, 1.0);
    vec4 highlight = vec4(min(main_color.rgb * 1.8 + 0.2, 1.0), 1.0);

    COLOR = mix(mix(shadow, main_color, smoothstep(0.0, 0.5, brightness)),
                highlight,
                smoothstep(0.5, 1.0, brightness));
}
```
