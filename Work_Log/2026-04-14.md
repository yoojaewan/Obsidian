## 확정된 설계 결정

### Head Group System
- 초상화 각도: 정면 고정
- 자세 바리에이션: `head.offset_y` (목 길이/거북목) + `head.rotation_z` (Z축 미세 기울기)
- Godot 씬 구조: Head Node2D 하위에 Face 파츠 + Helmet 포함, Hair/Armor는 그룹 밖
- Hair는 중력 표현을 위해 Head 그룹 제외 — 회전/오프셋 영향 없음
- Z 회전(갸웃)은 이미지 그대로 회전, 새 에셋 불필요

### Parts Metadata — 파츠 수치 시스템
- 각 파츠 JSON에 `beauty` (-10~+10), `age` (-10~+10) 수치 보유
- 유사 수치 파츠끼리 조합 → 자연스러운 인상 확보
- 캐릭터 총합 수치 = 장착 파츠 합산

### FaceTemplate
- 2개로 시작, 확장 구조 유지
- 귀 포함 (얼굴 실루엣 일부)
- 초상화 전용 — 필드 스프라이트 무관
- Template A: 넓고 둥근 얼굴 / Template B: 좁고 긴 얼굴
- 앵커 좌표 차이가 큰 두 형태로 시스템 확장성 검증

### Field Sprite 구조
- 공통 뼈대 사용 — 모든 캐릭터 동일
- 헤어/갑옷/투구로 시각적 구분
- 개성 표현: 기본자세(pose) 바리에이션
- FaceTemplate 영향 없음 (80×96에서 얼굴 디테일 무의미)
- 공격 모션: 무기 타입 기반 (별도 설계)

### 성별 에셋 분리 정책
- 분리: BodyType / FaceTemplate / Hair
- 공용: Eyes / Nose / Mouth / Armor / Helmet
- Eyes는 타입 선택 + scale/position으로 인상 조절 (속눈썹 유무는 타입으로 구분)
- Mouth: 1자입 스타일 — 성별 무관
- Beard: 남성 전용

### 초상화 구도
- 허리까지 표시 (머리 ~ 허리/벨트)
- 대화창: 얼굴 크롭 + 눈/입 파츠 교체로 감정 표현
- 이벤트 씬 등 용도별 크롭은 Godot 텍스처 영역 지정으로 처리

### Armor 시스템
- 통짜 스프라이트 (오버레이 없음, 추후 필요시 검토)
- 베이스: 바디타입 3 × 갑옷타입 3 = 9개
  - 갑옷 타입: 판금 / 가죽 / 천
  - 바디 타입: 덩치 / 기본 / 여성형
- 등급 차이: 색상으로만 처리 (초급=낡고 어두움, 고급=밝고 선명)
